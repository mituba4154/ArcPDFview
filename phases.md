> バージョン: 1.0.0  
> 作成日: 2026-03-01  
> 総工程目安: 12〜16 週間

---

## 工程サマリー

```
Phase 0 │ 基盤構築              │ 1〜2週  │ 環境・CI・プロジェクト骨格
Phase 1 │ コアPDF表示           │ 2〜3週  │ 開く・表示・ナビゲーション
Phase 2 │ 閲覧体験の完成        │ 2〜3週  │ 検索・目次・分割・ダークテーマ
Phase 3 │ 注釈機能              │ 2〜3週  │ ハイライト・コメント・手書き
Phase 4 │ 高度機能              │ 2〜3週  │ フォーム・印刷・ページ編集
Phase 5 │ 仕上げ・リリース      │ 1〜2週  │ パフォーマンス・配布パッケージ
```

## Phase 0 — 基盤構築

**期間目安:** 1〜2 週間  
**担当:** 環境構築は手動、骨格コードは Codex

### 目標
プロジェクト構造・依存関係・CI を整備し、空の Avalonia ウィンドウが起動できる状態にする。

### タスク一覧

- [ ] 0-1: .NET 8 SDK / Avalonia 11 の環境確認（Windows + Linux）
- [ ] 0-2: ソリューション構造の作成
  - `AcroPDF.Core` / `AcroPDF.Services` / `AcroPDF.ViewModels` / `AcroPDF.App` / `tests/`
- [ ] 0-3: NuGet パッケージの追加
  - `Avalonia` `Avalonia.Desktop` `Avalonia.Themes.Fluent`
  - `PDFium.NET` or `PdfiumViewer.Native`
  - `SkiaSharp.Views.Avalonia`
  - `CommunityToolkit.Mvvm`
  - `xUnit` `Moq`
- [ ] 0-4: Avalonia の空ウィンドウ起動確認
- [ ] 0-5: カラースキーム定義 (`Colors.axaml`) ← wireframe の CSS 変数をそのまま移植
- [ ] 0-6: `.github/copilot-instructions.md` の配置
- [ ] 0-7: GitHub Actions ワークフロー設定
  - `build.yml`: push 時に Win + Linux でビルド
  - `test.yml`: PR 時にテスト実行
- [ ] 0-8: `README.md` の作成

### 完了条件

- [ ] `dotnet build` がエラーなく通る
- [ ] Avalonia ウィンドウが Windows / Linux の両方で起動する
- [ ] GitHub Actions の初回ビルドが成功する

### 備考・注意点

> PDFium の NuGet はプラットフォームごとにネイティブバイナリが異なる。  
> `PDFium.NET.SDK` が最も安定しているが、ライセンスに商用利用制限がある場合は  
> `Sdcb.PdfiumBuild` (MIT) か `PdfiumViewer.Native` を検討すること。  
> Linux では `libpdfium.so` のパスを手動で設定する必要がある場合がある。

## Phase 1 — コア PDF 表示機能

**期間目安:** 2〜3 週間  
**依存:** Phase 0 完了

### 目標
PDF ファイルを開いて表示し、ページをめくれる最小の動作するビューワーを完成させる。

### タスク一覧

**PDF 読み込み**
- [ ] 1-1: `IPdfRenderService` インターフェース実装
- [ ] 1-2: `PdfiumRenderService` の実装（PDFium でファイルを開く）
- [ ] 1-3: `PdfDocument` / `PdfPage` モデルの実装
- [ ] 1-4: パスワード付き PDF の対応（入力ダイアログ）
- [ ] 1-5: ファイルドラッグ&ドロップ対応
- [ ] 1-6: 起動引数からファイルを開く対応

**ページレンダリング**
- [ ] 1-7: `PdfPageControl`（SkiaSharp カスタムコントロール）の作成
- [ ] 1-8: 指定 DPI でページを `SKBitmap` にレンダリング
- [ ] 1-9: ページキャッシュの実装（`ConcurrentDictionary<int, SKBitmap>`）
- [ ] 1-10: ズーム機能（25%〜400%）
- [ ] 1-11: Ctrl+スクロールでズーム
- [ ] 1-12: 「幅に合わせる」「ページに合わせる」「実寸」

**ページナビゲーション**
- [ ] 1-13: ツールバーの前/次/先頭/最終ボタン
- [ ] 1-14: ページ番号入力ジャンプ
- [ ] 1-15: キーボードショートカット（← → Home End）
- [ ] 1-16: 連続スクロールモード（全ページ縦並び）
- [ ] 1-17: 単ページモード

**タブ管理**
- [ ] 1-18: `TabViewModel` の実装
- [ ] 1-19: タブバー UI（アクティブ・非アクティブ・閉じるボタン）
- [ ] 1-20: 「+」ボタンで新しいファイルを開く
- [ ] 1-21: タブの右クリックメニュー
- [ ] 1-22: Ctrl+W でタブを閉じる
- [ ] 1-23: 最後のタブを閉じてもアプリは終了しない（空の状態を表示）

**左サイドバー — サムネイル**
- [ ] 1-24: サムネイルパネルの基本 UI
- [ ] 1-25: 非同期でサムネイル生成（バックグラウンド）
- [ ] 1-26: サムネイルクリックでページジャンプ
- [ ] 1-27: 現在ページのサムネイルをハイライト（自動スクロール）

**ステータスバー**
- [ ] 1-28: 現在ページ / 総ページ / ズーム率の表示
- [ ] 1-29: マウス座標の表示

### 完了条件

- [ ] PDF を開いて全ページ表示できる
- [ ] ズーム・ページ移動が正常に動作する
- [ ] 3 ファイルをタブで同時に開ける
- [ ] サムネイルサイドバーが動作する

### 備考・注意点

> `PdfPageControl` は Avalonia の `SKCanvasView` を継承して実装する。  
> 大きい PDF（500 ページ以上）では連続スクロールモードに仮想化（表示範囲外はレンダリングしない）が必要。  
> サムネイル生成は低 DPI（48px 幅相当）で行い、`Task.Run` + `SemaphoreSlim` で並列数を制限すること（3〜4 並列まで）。

> テキスト選択の実装は PDFium の `FPDFText_*` API を使う。

> `SKCanvas` の描画ループ内で重ね書きする。

**フリーハンド**

### 備考・注意点

> PDFium での注釈書き込みは `FPDF_Annot_*` API グループ。  
> 注釈の座標系は PDF の座標系（左下原点）なので、画面座標（左上原点）との変換に注意。  
> `AnnotationOverlay` コントロールは `PdfPageControl` の上に透明な Avalonia レイヤーとして重ねる設計にする。
